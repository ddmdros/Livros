Variables

Assigning variables

The first time you declare a variable, you use one of these:

  • let: variables that might have a new value.
    • let value = 3;
    • let str = "some string";
  • const: used to variables that are assigned oce (pi value, for instance).
    • const PI_VALUE = 3.14159;
    • It cannot be changed later without throwing an error: Uncaught TypeError: Assignment to constant variable.
  • var: the difference between let and var is related to scope, but keeping it simple: it's a legacy code that we don't use anymore.
    • var has global scope and let has a {block scope}.

After that, you only use the name of the existing variable to assign a new value:

  • let firstname = "Bernardo";
  • firstname = "Diogo";

Naming variables

Variables start with a lowercase letter (not a number) and should be descriptive

Cannot contain space, but underscore is allowed

  • ageOfBuyer (camelCase) -- OK and used withing the book
  • age_of_buyer (snake_case) -- OK
  • AGE_OF_BUYER (SCREAMING_CASE) -- for const
  • When choosing is allowed, consistency is key

Primitive data types

JS has seven primitive types:

  1. String:
    • let message = `Let's learn ${language}`;
    • Escape characters:
      • "Hello, what's your name? It is \"Diogo\"?"
  2. Number: integer, floating point, exponential, octal, hexa and binary
  3. BigInt: those have a postfix n: 90071992574409920n
  4. Boolean: true or false
  5. Symbol: they can have the same value and be different
  6. Undefined: Type for variables that have not been assigned a value.
    • DON'T do this: let terribleThingToDo: undefined;
  7. null: the value is empty or has an unknown value.
    • You can do this: let betterOption = null;
